# Variables, Types, and Control Flow in PHP

## บทนำ

หัวข้อนี้คือรากฐานของการเขียน PHP เพราะไม่ว่าคุณจะพัฒนาเว็บ, API, หรือระบบหลังบ้าน คุณต้องเข้าใจว่า **ข้อมูลถูกเก็บอย่างไร**, **มีชนิดข้อมูลแบบไหน**, และ **โปรแกรมตัดสินใจหรือวนลูปอย่างไร**

บทเรียนนี้เหมาะกับผู้เริ่มต้นและคนที่อยากทบทวนพื้นฐานแบบ modern โดยจะอิงแนวคิดที่ใช้จริงใน PHP ยุคปัจจุบัน เช่น type declarations, `strict_types`, การใช้ `===` แทน `==` เมื่อเหมาะสม, และ `match` สำหรับงานที่ต้องเลือกเงื่อนไขจากค่าหลายแบบอย่างอ่านง่ายขึ้น

## สิ่งที่ผู้เรียนจะได้เรียน

- การประกาศตัวแปรและการตั้งชื่อแบบอ่านง่าย
- ชนิดข้อมูลพื้นฐานใน PHP
- ความต่างระหว่าง `==` และ `===`
- การใช้ `if`, `elseif`, `else`, `switch`, `match`
- การวนลูปด้วย `foreach` และ `for`
- แนวทางเขียนโค้ดให้ปลอดภัยและดูแลง่ายขึ้น

## หัวข้อย่อยที่ 1: Variables ใน PHP คืออะไร

ตัวแปรใน PHP ใช้เครื่องหมาย `$` นำหน้าชื่อ เช่น `$name`, `$price`, `$isActive` และเป็นจุดเริ่มต้นของการเก็บข้อมูลทุกชนิดในโปรแกรม

แนวทางที่ดีคือควรตั้งชื่อตัวแปรให้สื่อความหมาย เช่น `$fullName` ดีกว่า `$x` เพราะช่วยให้โค้ดอ่านง่ายและดูแลต่อได้ง่ายกว่า โดยเฉพาะเมื่อโปรเจกต์เริ่มใหญ่ขึ้น

```php
<?php

declare(strict_types=1);

// ตัวแปรสำหรับเก็บข้อมูลพื้นฐาน
$fullName = "Phumin";
$age = 25;
$isMember = true;

// แสดงผลข้อมูล
echo $fullName . PHP_EOL;
echo $age . PHP_EOL;
echo $isMember ? "member" : "guest";
```

### สิ่งที่ควรเข้าใจ

- PHP ใช้ `$` นำหน้าชื่อตัวแปร
- ตัวแปรเดียวกันอาจเก็บค่าคนละชนิดได้ แต่ในการเขียนงานจริงควรทำให้ชัดเจน
- การตั้งชื่อที่ดีช่วยลดความสับสนในระยะยาว

---

## หัวข้อย่อยที่ 2: Types พื้นฐานที่ใช้บ่อย

PHP มีชนิดข้อมูลหลายแบบ เช่น `bool`, `int`, `float`, `string`, `array`, `object`, `null` และชนิดอื่น ๆ อีกหลายแบบในระบบภาษา

สำหรับ `array` ใน PHP ไม่ได้เป็นเพียง list อย่างเดียว แต่ใช้ได้ทั้งแบบ list, key-value, หรือโครงสร้างข้อมูลที่ซับซ้อนขึ้นได้ จึงเป็นชนิดข้อมูลที่ใช้บ่อยมากในงานจริง

```php
<?php

declare(strict_types=1);

// scalar types
$name = "Notebook";   // string
$stock = 10;          // int
$price = 59.90;       // float
$isAvailable = true;  // bool

// array แบบ list
$tags = ["php", "backend", "web"];

// array แบบ key-value
$product = [
    "name" => $name,
    "stock" => $stock,
    "price" => $price,
    "available" => $isAvailable,
];

var_dump($product);
```

### สิ่งที่ควรเข้าใจ

- `string`, `int`, `float`, `bool` คือชนิดพื้นฐานที่ใช้บ่อย
- `array` ใช้ได้ทั้งเก็บลำดับข้อมูลและ key-value
- `var_dump()` เหมาะกับการดูชนิดข้อมูลและค่าจริงระหว่างฝึก

---

## หัวข้อย่อยที่ 3: Type Declarations และ `strict_types`

ใน PHP สามารถกำหนดชนิดข้อมูลให้กับ parameter และ return type ได้ และสามารถเปิด `declare(strict_types=1);` เพื่อให้การตรวจ scalar types เข้มงวดขึ้นในระดับไฟล์

การใช้ type declarations ทำให้โค้ดชัดเจนขึ้น และช่วยจับข้อผิดพลาดเร็วขึ้น โดยเฉพาะในโปรเจกต์ที่ต้องการความน่าเชื่อถือ

```php
<?php

declare(strict_types=1);

// ฟังก์ชันนี้รับ int 2 ค่า และคืนค่า int
function add(int $a, int $b): int
{
    return $a + $b;
}

echo add(10, 20);
```

ตัวอย่างกรณีที่ควรระวัง:

```php
<?php

declare(strict_types=1);

function multiply(int $a, int $b): int
{
    return $a * $b;
}

// strict_types เปิดอยู่ จึงไม่ควรส่ง string แทน int
echo multiply("5", 4);
```

### สิ่งที่ควรเข้าใจ

- `strict_types` ช่วยลดความคลุมเครือของ type coercion
- งานจริงควรใช้ type hints ให้มากเท่าที่เหมาะสม
- โค้ดจะอ่านง่ายขึ้นเมื่อ function บอกชัดว่ารับอะไรและคืนอะไร

---

## หัวข้อย่อยที่ 4: Type Juggling และความต่างของ `==` กับ `===`

PHP มีพฤติกรรมเรื่อง type juggling ในหลายบริบท จึงต้องระวังเวลาเปรียบเทียบค่า โดย `==` คือการเทียบแบบหลวมที่อาจมีการแปลงชนิดข้อมูลก่อน ส่วน `===` คือการเทียบทั้งค่าและชนิดข้อมูลพร้อมกัน

ในงานจริงควรเลือกใช้ `===` เป็นค่าเริ่มต้นเมื่อคุณต้องการความชัดเจน

```php
<?php

declare(strict_types=1);

var_dump(10 == "10");   // true
var_dump(10 === "10");  // false

$status = "1";

// แนะนำให้ใช้ === เมื่อรู้ชนิดข้อมูลที่คาดหวัง
if ($status === "1") {
    echo "Active";
}
```

### สิ่งที่ควรเข้าใจ

- `==` อาจทำให้เกิดผลลัพธ์ที่ไม่คาดคิด
- `===` ปลอดภัยกว่าเมื่อคุณต้องการความแน่นอน
- ในเงื่อนไขสำคัญ เช่น auth, role, status ควรระวังเรื่องชนิดข้อมูลเสมอ

---

## หัวข้อย่อยที่ 5: Boolean และเงื่อนไขที่ใช้ใน `if`

ชนิด `bool` มีแค่ `true` และ `false` และผลลัพธ์จาก operators จำนวนมากจะถูกส่งต่อไปใช้ใน control structures เช่น `if`

โครงสร้าง `if`, `elseif`, `else` เหมาะกับกรณีที่เงื่อนไขมีตรรกะหลายชั้นและไม่ได้พิจารณาจากค่าเดียวกันทั้งหมด

```php
<?php

declare(strict_types=1);

$score = 82;

if ($score >= 80) {
    echo "Grade A";
} elseif ($score >= 70) {
    echo "Grade B";
} else {
    echo "Need improvement";
}
```

### สิ่งที่ควรเข้าใจ

- `if` ใช้เมื่อมีเงื่อนไขเชิงตรรกะ
- `elseif` ช่วยแยกหลายกรณีให้ชัด
- ควรเขียนเงื่อนไขให้อ่านแล้วเข้าใจทันที

---

## หัวข้อย่อยที่ 6: `switch` และ `match`

`switch` เหมาะกับกรณีที่ต้องเปรียบเทียบค่าเดียวกับหลายทางเลือก แต่ `match` ซึ่งมีตั้งแต่ PHP 8 จะอ่านง่ายขึ้นในหลายกรณี เพราะคืนค่าได้ตรง ๆ และลดความผิดพลาดจากการลืม `break`

สำหรับโค้ด modern PHP ถ้าคุณต้อง mapping ค่าหนึ่งไปยังอีกค่าหนึ่ง เช่น status code ไปยังข้อความ `match` มักเหมาะกว่า

### ตัวอย่าง `switch`

```php
<?php

declare(strict_types=1);

$role = "admin";

switch ($role) {
    case "admin":
        echo "Full access";
        break;
    case "editor":
        echo "Edit access";
        break;
    default:
        echo "Read only";
}
```

### ตัวอย่าง `match`

```php
<?php

declare(strict_types=1);

$role = "editor";

// match คืนค่าได้ตรง ๆ
$message = match ($role) {
    "admin" => "Full access",
    "editor" => "Edit access",
    default => "Read only",
};

echo $message;
```

### สิ่งที่ควรเข้าใจ

- `switch` ใช้ได้ดี แต่ต้องระวัง `break`
- `match` อ่านง่ายกว่าในหลายสถานการณ์
- `match` เหมาะมากกับงาน mapping ค่า

---

## หัวข้อย่อยที่ 7: การวนลูปด้วย `foreach`

`foreach` เป็นวิธีที่ง่ายและเหมาะมากสำหรับการวนข้อมูลใน array

สำหรับงานทั่วไป เช่น แสดงรายการสินค้า, รายชื่อพนักงาน, หรือผลลัพธ์จากฐานข้อมูลที่แปลงเป็น array แล้ว `foreach` มักเป็นตัวเลือกแรก

```php
<?php

declare(strict_types=1);

$users = [
    ["name" => "Anna", "role" => "admin"],
    ["name" => "Ben", "role" => "editor"],
    ["name" => "Cat", "role" => "viewer"],
];

foreach ($users as $user) {
    // อ่านค่าจาก array ทีละรายการ
    echo $user["name"] . " - " . $user["role"] . PHP_EOL;
}
```

ตัวอย่างดึง key และ value พร้อมกัน:

```php
<?php

declare(strict_types=1);

$settings = [
    "app_name" => "PHP Demo",
    "debug" => true,
];

foreach ($settings as $key => $value) {
    echo $key . ": ";
    var_dump($value);
}
```

### สิ่งที่ควรเข้าใจ

- `foreach` ใช้ง่ายและอ่านง่าย
- เหมาะกับ array มากที่สุด
- ใช้ `$key => $value` เมื่ออยากได้ชื่อ key ด้วย

---

## หัวข้อย่อยที่ 8: การวนลูปด้วย `for`

`for` เหมาะกับกรณีที่คุณรู้จำนวนรอบล่วงหน้า หรืออยากควบคุมตัวนับอย่างชัดเจน เช่น สร้างเลขลำดับ, ทำ pagination logic บางส่วน, หรือวนตาม index

```php
<?php

declare(strict_types=1);

// วนลูป 5 รอบ
for ($i = 1; $i <= 5; $i++) {
    echo "Round: {$i}" . PHP_EOL;
}
```

### สิ่งที่ควรเข้าใจ

- `for` เหมาะเมื่อรู้จำนวนรอบชัดเจน
- ควรตั้งเงื่อนไขให้ชัดว่าลูปจะหยุดเมื่อไร
- ถ้าเป็นการอ่านสมาชิกใน array โดยตรง ส่วนใหญ่ `foreach` อ่านง่ายกว่า

---

## หัวข้อย่อยที่ 9: ตัวอย่างใช้งานจริงแบบรวมทุกแนวคิด

ตัวอย่างนี้รวมตัวแปร, types, comparison, `match`, และ `foreach` เข้าด้วยกันในสถานการณ์ที่ใกล้งานจริง เช่น การสรุปสถานะคำสั่งซื้อ

```php
<?php

declare(strict_types=1);

$orders = [
    ["id" => 1001, "amount" => 1500.00, "status" => "paid"],
    ["id" => 1002, "amount" => 0.00, "status" => "pending"],
    ["id" => 1003, "amount" => 2300.50, "status" => "cancelled"],
];

foreach ($orders as $order) {
    // แปลง status เป็นข้อความที่อ่านง่าย
    $label = match ($order["status"]) {
        "paid" => "ชำระเงินแล้ว",
        "pending" => "รอดำเนินการ",
        "cancelled" => "ยกเลิก",
        default => "ไม่ทราบสถานะ",
    };

    // ใช้ comparison แบบชัดเจน
    if ($order["amount"] > 0) {
        echo "Order #{$order['id']} : {$label} ({$order['amount']} THB)" . PHP_EOL;
    } else {
        echo "Order #{$order['id']} : {$label} (ยังไม่มีมูลค่า)" . PHP_EOL;
    }
}
```

### สิ่งที่ควรเข้าใจ

- ใช้ array เก็บข้อมูลหลายรายการ
- ใช้ `foreach` วนอ่านทีละรายการ
- ใช้ `match` แปลงค่าภายในระบบให้เป็นข้อความที่อ่านง่าย
- ใช้ `if` สำหรับตัดสินใจตามเงื่อนไขเพิ่มเติม

## หัวข้อย่อยที่ 10: Best Practices ที่ควรใช้ตั้งแต่ต้น

ในการเขียน PHP ยุคปัจจุบัน ควรใช้ `declare(strict_types=1);` ในไฟล์แอปที่เหมาะสม, ใช้ type declarations ให้ชัด, เลือก `===` เป็นค่าเริ่มต้นเมื่อต้องเทียบค่าจริงจัง, และใช้ `match` เมื่อโค้ดต้อง map ค่าแบบตรงไปตรงมา

### สรุปแนวทางสั้น ๆ

- ตั้งชื่อตัวแปรให้สื่อความหมาย
- ใช้ type hints เมื่อทำได้
- ใช้ `===` ก่อน `==`
- ใช้ `foreach` กับ array
- ใช้ `match` เมื่อโค้ดเป็นการเลือกผลลัพธ์จากค่าหนึ่งค่า
